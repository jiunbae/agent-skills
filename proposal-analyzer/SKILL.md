# Proposal Analyzer (제안서 분석기)

사업 제안서 및 RFP 문서를 분석하여 사업 진행 타당성을 평가하는 스킬입니다.

## Instructions

사용자가 제안서/RFP 문서 분석을 요청하면 다음 단계를 수행하세요:

### 1. 문서 읽기
- Read 도구를 사용하여 제공된 문서를 읽습니다
- PDF, Word, 마크다운 등 다양한 형식을 지원합니다

### 2. 핵심 정보 추출
문서에서 다음 정보를 추출하세요:

**기본 정보**
- 사업명
- 발주처
- 사업 개요

**일정 정보**
- 제안서 제출 마감일
- 사업 수행 기간
- 주요 마일스톤

**예산 정보**
- 총 사업 예산
- 예산 구성 (인건비, 장비, 기타)

**기술 요구사항**
- 필수 기술 스택
- 기능 요구사항
- 성능 요구사항
- 보안 요구사항

**자격 요건**
- 참가 자격
- 필수 인증/자격증
- 유사 사업 경험

### 3. 적정성 분석

각 항목에 대해 다음 기준으로 평가하세요:

#### 가격 적정성
- 업계 평균 대비 예산 수준
- 인건비 단가의 합리성 (MM당 금액)
- 숨은 비용 또는 리스크 요소

#### 기한 적정성
- 요구사항 대비 기간의 충분성
- 준비 기간의 적절성
- 일정 리스크 요소

#### 기술 스펙 적정성
- 기술적 구현 난이도
- 필요 인력 수준
- 기술 리스크 요소

## Examples

### 예시 1: PDF 제안서 분석 요청
```
사용자: prop.pdf 파일을 분석해서 사업 참여할만한지 봐주세요
→ Read 도구로 PDF 읽기 → 핵심 정보 추출 → 적정성 분석 → 보고서 생성
```

### 예시 2: 특정 관점 분석 요청
```
사용자: 이 RFP의 기술 요구사항이 우리 팀이 감당할 수 있는 수준인지 분석해주세요
→ 기술 스펙에 집중한 심층 분석 수행
```

## 분석 시 참고 기준

### 소프트웨어 개발 인건비 기준 (2024년 한국 기준)
- 특급 기술자: 월 1,200만원 이상
- 고급 기술자: 월 900-1,200만원
- 중급 기술자: 월 600-900만원
- 초급 기술자: 월 400-600만원

### 일반적인 프로젝트 기간 추정
- 단순 웹사이트: 1-2개월
- 중규모 웹 애플리케이션: 3-6개월
- 대규모 시스템: 6-12개월
- 복잡한 플랫폼: 12개월 이상

### 수익성 기준
- 최소 마진: 15-20%
- 권장 마진: 25-30%
- 고위험 프로젝트: 35% 이상
