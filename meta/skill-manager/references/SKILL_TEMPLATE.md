## 섹션별 가이드

### 1. YAML Frontmatter (필수)

```yaml
---
name: kebab-case-name      # 필수: 스킬 식별자
description: 설명           # 필수: 활성화 키워드 포함
---
```

**작성 팁:**
- `name`: 디렉토리명과 동일하게 kebab-case 사용
- `description`: 한국어로 작성, 활성화 키워드를 자연스럽게 포함

### 2. Overview (필수)

스킬의 목적을 명확히 설명합니다.

**포함 요소:**
- 스킬이 해결하는 문제
- 핵심 기능 목록 (불릿 포인트)
- 지원 범위/제한

**좋은 예:**
```markdown
## Overview

코드 리뷰 프로세스를 체계화하는 스킬입니다.

**핵심 기능:**
- 코드 품질 체크리스트 제공
- 보안 취약점 검사
- 성능 이슈 탐지
```

**나쁜 예:**
```markdown
## Overview

코드 리뷰를 합니다.
```

### 3. When to Use (필수)

활성화 조건을 구체적으로 명시합니다.

**포함 요소:**
- 명시적 요청 키워드
- 자동 활성화 조건

**좋은 예:**
```markdown
## When to Use

**명시적 요청:**
- "코드 리뷰해줘"
- "이 PR 검토해줘"
- "코드 품질 확인해줘"

**자동 활성화:**
- git diff 결과가 있을 때
- PR 관련 작업 중
```

### 4. Workflow (필수)

단계별 실행 절차를 상세히 기술합니다.

**포함 요소:**
- 명확한 단계 구분 (Step 1, 2, 3...)
- 각 단계의 목적 설명
- 코드/명령어 예시

**팁:**
- 복잡한 워크플로우는 Mode로 구분 (Mode 1, Mode 2...)
- 의사결정 포인트 명시
- 입출력 예시 포함

### 5. Examples (필수)

최소 2개 이상의 구체적 사용 예시를 제공합니다.

## 스타일 가이드

### 언어
- 설명: 한국어
- 코드/명령어: 영어
- 변수/파일명: 영어

### 포맷팅
- 제목: `## 섹션명` (H2)
- 하위 제목: `### 서브섹션명` (H3)
- 코드 블록: 언어 명시 (\`\`\`python)
- 강조: **굵게**, `코드`

### 구조
- 불릿 포인트: `-` 사용
- 번호 목록: 순서가 중요할 때만
- 테이블: 비교, 목록에 활용

### 길이
- Overview: 5-10줄
- 각 Workflow Step: 10-20줄
- 각 Example: 10-30줄
- Best Practices: 각 3-5항목
