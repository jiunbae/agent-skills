# AIR specification sources

The normative project specification is [AIR-1.0.0.md](AIR-1.0.0.md).

Stable public identifiers map to the checked-in sources as follows:

| Public identifier | Repository source |
|---|---|
| `/air/spec/1.0.0/` | `spec/AIR-1.0.0.md` |
| `/air/schema/1.0.0/air.schema.json` | `schemas/air.schema.json` |
| `/air/schema/1.0.0/workflow.schema.json` | `schemas/air-workflow.schema.json` |
| `/air/schema/1.0.0/plan.schema.json` | `schemas/air-plan.schema.json` |
| `/air/schema/1.0.0/trace.schema.json` | `schemas/air-trace.schema.json` |
| `/air/schema/1.0.0/problem.schema.json` | `schemas/air-problem.schema.json` |
| `/air/v1` | `schemas/air.openapi.json` |

These URLs are identifiers in offline validation. Network retrieval is not
required. Publishing them at the project-controlled origin is a separate
release operation.
