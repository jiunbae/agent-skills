# Codex 스킬 연동 메모

Codex는 `SKILL.md` 기반 스킬을 기본 지원합니다. 이 파일은 스킬 선택
규칙을 다시 정의하지 않으며, 설치 관리자가 참고할 호환성 정보만
기록합니다.

## 지원 경로

- 사용자 전역 스킬: `~/.agents/skills/<skill>/SKILL.md`
- 저장소 스킬: 저장소 루트 또는 현재 디렉터리까지의
  `.agents/skills/<skill>/SKILL.md`
- 시스템 스킬: Codex가 자체 관리

스킬 폴더 심볼릭 링크가 지원되므로, 이 저장소의 스킬 디렉터리를 위
경로에 개별 링크합니다. `~/.codex/skills` 전체를 교체하거나 그 안의
`.system` 디렉터리를 이동하지 않습니다.

## 작성 원칙

- `description`에 명확한 호출 범위와 제외 조건을 둡니다.
- 명시적으로 선택된 스킬의 `SKILL.md`는 끝까지 읽습니다.
- references, scripts, assets는 해당 작업에 필요할 때만 로드합니다.
- 스킬 목록을 매 작업마다 셸로 다시 만들거나 응답에 `[skill: none]`을
  강제로 출력하지 않습니다.

## 엔지니어링 기본 원칙

- 구현 전에 결과에 영향을 주는 가정과 불확실성을 드러냅니다.
- 요청을 충족하는 가장 단순한 설계를 우선하고 미요청 기능과 추상화를
  추가하지 않습니다.
- 변경 범위를 요청과 직접 연결되는 코드로 제한하고 주변 코드를 임의로
  정리하지 않습니다.
- 완료 조건을 실행 가능한 테스트나 검사로 정의하고 변경 후 실제로
  검증합니다.
- 설계 결정이 필요한 변경, 대규모 리팩터링, 새 운영 의존성은 사용자와
  합의한 뒤 진행합니다.
